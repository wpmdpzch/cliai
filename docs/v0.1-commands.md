# CLI-AI v0.1 MVP 内置命令清单

> 版本目标：2026-Q2
> 核心功能：用自然语言执行简单系统命令

---

## 设计原则

1. **MVP 优先**：先实现最高频的命令，框架先跑通
2. **Go 原生**：高频核心命令用 Go 实现（curl, jq, base64 等）
3. **系统兜底**：其他命令包装系统调用，快速完成 MVP

## 命令分类

### P0 - 必须实现（v0.1 MVP）

```
network: curl
text: jq, grep, cat
file: ls, head, tail
system: ps, df, free
encoding: base64
```

### P1 - 应该实现（v0.1 完成）

```
network: wget, ping
text: sed, awk, cut, sort, wc
file: find, du

```

### P2 - 尽量实现（v0.1）

```
network: nc
```

## 实现方式

### Go 原生命令（需要自己写）

| 命令 | 原因 | 工作量 |
|------|------|--------|
| curl | 核心 HTTP 工具，跨平台一致 | 高 |
| jq | JSON 处理，复杂逻辑 | 高 |
| base64 | 简单算法 | 低 |
| md5 | 标准算法 | 低 |
| sha256 | 标准算法 | 低 |

### 系统调用命令（包装系统命令）

| 命令 | 系统命令 | 包装工作量 |
|------|---------|-----------|
| grep | /bin/grep | 低 |
| sed | /bin/sed | 低 |
| awk | /usr/bin/awk | 低 |
| cat | /bin/cat | 低 |
| ls | /bin/ls | 低 |
| ... | ... | 低 |

## 命令元数据格式

每个命令在 `commands.json` 中声明：

```json
{
  "name": "curl",
  "category": "network",
  "description": "发送 HTTP/HTTPS 请求",
  "usage": "curl [options] <url>",
  "example": "curl https://api.example.com/ping",
  "implemented": "go",
  "dangerous": false,
  "version": "1.0.0"
}
```
